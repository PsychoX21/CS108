Debugging Flags

  • -g → Includes debugging symbols for use with gdb.

Sanitizers (Runtime Error Detection)

  • -fsanitize=address → Detects memory corruption issues like buffer overflows and use-after-free.
  • -fsanitize=undefined → Detects undefined behavior (like signed integer overflow, invalid shifts).
  • -fsanitize=leak → Detects memory leaks.

  • -Wall → Enables most common warnings.

  • -Werror → Treats all warnings as errors (prevents compiling if warnings exist).
  • -Wshadow → Warns if a variable shadows another variable.

Sure! Here’s a detailed breakdown of various g++ compilation flags you can use for debugging, warnings, sanitization, and optimizations.

1. Debugging Flags 🛠️

These help with debugging using gdb:

  • -g → Generates debug symbols for use with gdb (does not affect optimization).
  • -ggdb → More detailed debugging information tailored for gdb.

Example:

sh
g++ -g my_program.cpp -o my_program

Now, you can run:

sh
gdb ./my_program

2. Warning and Error Flags ⚠️

These help catch potential issues in your code.

Basic Warnings

  • -Wall → Enables most common warnings.
  • -Wextra → Enables additional warnings beyond -Wall.
  • -Wpedantic → Warns about non-standard C++ code.

Strict Error Handling

  • -Werror → Treats all warnings as errors, stopping compilation.
  • -Wfatal-errors → Stops at the first error instead of continuing.

Common Specific Warnings

  • -Wshadow → Warns if a variable name shadows another variable.
  • -Wconversion → Warns about implicit conversions that may cause issues.
  • -Wfloat-equal → Warns about direct floating-point equality comparisons (==).
  • -Wsign-compare → Warns when signed and unsigned integers are compared.
  • -Wformat → Warns about mismatches in printf/scanf formats.
  • -Wunused-variable → Warns about unused variables.
  • -Wuninitialized → Warns about using uninitialized variables.
  • -Wdeprecated → Warns about deprecated features.

Example:

sh
g++ -Wall -Wextra -Wpedantic -Werror my_program.cpp -o my_program

If there’s a warning, compilation fails due to -Werror.

3. Sanitizers (Runtime Error Detection) 🧪

These flags instrument your code at runtime to catch issues.

Memory Error Detection

  • -fsanitize=address → Detects buffer overflows, use-after-free, and memory leaks.
  • -fsanitize=leak → Specifically detects memory leaks.

Undefined Behavior Detection

  • -fsanitize=undefined → Detects undefined behavior (e.g., invalid shifts, signed overflows).
  • -fsanitize=bounds → Detects out-of-bounds array access.

Thread and Concurrency Issues

  • -fsanitize=thread → Detects race conditions in multithreaded code.

Example Usage

sh
g++ -g -fsanitize=address -fsanitize=undefined my_program.cpp -o my_program
./my_program

If there’s an error, the sanitizer prints a runtime message showing the issue.

4. Optimization Flags 🚀

These control compiler optimizations:

  • -O0 → No optimization (default, useful for debugging).
  • -O1 → Basic optimizations.
  • -O2 → More optimizations, good for performance.
  • -O3 → Aggressive optimizations (may change floating-point behavior).
  • -Og → Optimized for debugging (recommended for debugging instead of -O0).

Example:

sh
g++ -g -Og my_program.cpp -o my_program

This gives good debugging info without hurting performance.

5. Miscellaneous Useful Flags 🔧

  • -std=c++17 or -std=c++20 → Use a specific C++ standard.
  • -fno-exceptions → Disables exception handling.
  • -fno-rtti → Disables Run-Time Type Information (reduces binary size).
  • -march=native → Optimizes code for the CPU running the compiler.

Example:

sh
g++ -std=c++20 -Wall -Wextra -Werror -O2 -march=native my_program.cpp -o my_program

This enables C++20, strict warnings, optimizations, and CPU-specific tuning.

6. Putting It All Together

For debugging and error checking, a good compilation command is:

sh
g++ -g -Wall -Wextra -Wpedantic -Wshadow -Wconversion -Werror -fsanitize=address -fsanitize=undefined -O0 my_program.cpp -o my_program

For optimized release builds:

sh
g++ -std=c++20 -O3 -march=native -Wall -Wextra -Wpedantic -Werror my_program.cpp -o my_program
